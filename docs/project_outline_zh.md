# 项目大纲（中文）

## 1. 适用对象

面向对优化、运筹学、人工智能与物流系统感兴趣的本科生。

## 2. 先修要求

- 具备 Python 编程能力
- 了解基础优化概念（线性/整数规划更佳）
- 具备基础概率统计知识
- 具备实验可复现意识

## 3. 学习目标

完成本项目后，学生应能够：

1. 解释 VRP、CVRP、EVRP-TW 与卡车-无人机协同问题。
2. 复现至少一个基线求解器或策略模型。
3. 使用清晰指标进行性能与可行性评估。
4. 在真实约束下分析失败案例。
5. 在报告中提供可复现的技术证据。

## 4. 八周自驱计划

### 第 1 周：环境与入门
- 环境搭建与仓库协作流程
- 数据格式理解
- 产出：环境清单与 smoke run 成功记录

### 第 2 周：基础与文献梳理
- 阅读核心 VRP/EVRP 文献
- 梳理约束与目标的概念图
- 产出：阅读备忘录

### 第 3 周：基线复现
- 复现一个基线（OR-Tools 启发式或 RL 基线）
- 验证目标值与可行性输出
- 产出：基线运行日志

### 第 4 周：EVRP-TW 专题
- 实现或运行 EVRP-TW 实验
- 记录电量与时间窗违约情况
- 产出：指标表格

### 第 5 周：空地协同建模
- 定义卡车-无人机协同 EVRP-TW 模型
- 明确同步与协同约束
- 产出：建模说明文档

### 第 6 周：混合求解实现
- 实现启发式/RL/混合方法
- 在小规模基准上测试
- 产出：演示结果

### 第 7 周：改进与消融
- 优化一个模块（路径、充电、任务分配）
- 与基线进行公平对比
- 产出：消融与对比表

### 第 8 周：最终集成
- 最终报告与可复现提交包
- 演示汇报
- 产出：终版提交

## 5. 评价建议（非授课模式）

- 25% 可复现性与工程质量
- 25% 基线与改进方法对比质量
- 20% 失败/误差分析深度
- 30% 最终报告与演示质量

## 6. 每周推进节奏

- 周初：定义可量化目标
- 周中：实现并进行可控实验
- 周末：总结结果与风险项
- 遇阻塞时可选导师沟通

## 7. 最终提交要求

- 源代码与运行脚本
- 配置文件与随机种子
- 关键实验日志/图表
- 最终报告（问题、方法、结果、局限）
- 演示视频（5-8 分钟）
