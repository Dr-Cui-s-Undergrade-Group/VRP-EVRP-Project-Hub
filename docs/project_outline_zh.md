# 项目大纲：VRP、EVRP-TW 与卡车-无人机协同路径优化

## 1. 适用对象

本 SEP 暑研项目面向对优化、运筹学、物流系统、人工智能或组合优化强化学习感兴趣的本科生。

项目核心问题是：在容量、时间窗、电池、充电站以及卡车-无人机同步等真实约束下，路径算法如何做出可行且高质量的决策。

## 2. 先修要求

建议学生具备：

- Python 编程能力；
- 基础概率统计知识；
- 认真阅读论文和记录实验的意愿；
- 基础算法理解能力。

以下能力有帮助，但不是硬性要求：

- 线性规划或整数规划；
- 图算法；
- PyTorch；
- 强化学习；
- 运筹学基础。

## 3. 学习目标

完成本项目后，学生应能够：

1. 解释 CVRP、VRPTW、EVRP-TW 和卡车-无人机协同路径问题。
2. 复现一个基线求解器或学习型策略。
3. 同时报告目标值、可行性、约束违约和运行时间。
4. 诊断路线为什么不可行或效率低。
5. 实现一个聚焦改进或约束扩展。
6. 用清晰证据展示一个可复现的优化实验。

## 4. 研究方向

| 方向 | 核心问题 | 常见工具 |
|---|---|---|
| 经典 VRP / VRPTW | 标准求解器如何处理容量和时间窗？ | OR-Tools、PyVRP |
| EVRP-TW 深度强化学习 | 学习型策略能否构造可行的电动车路线？ | POMO、Attention Model、RL4CO |
| 混合 EVRP-TW | 学习方法与局部搜索能否互相修复或增强？ | PyVRP、自定义修复启发式、神经构造策略 |
| 卡车-无人机协同 | 地面车辆和无人机如何同步服务？ | 小规模自定义算例、启发式搜索、makespan 分析 |

学生应从简单基线开始，再逐步增加电池或无人机约束。

## 5. 八周计划

| 周次 | 重点 | 产出 |
|---|---|---|
| 第 1 周 | 环境与基线搭建 | 一个小算例求解记录 |
| 第 2 周 | 问题变体与文献理解 | 阅读笔记与约束图 |
| 第 3 周 | 基线复现 | 目标值、可行性、运行时间日志 |
| 第 4 周 | EVRP-TW 约束 | 时间窗与电量违约表 |
| 第 5 周 | 卡车-无人机建模 | 建模说明与同步规则 |
| 第 6 周 | 混合或学习方法 | 小规模算例演示运行 |
| 第 7 周 | 改进与消融 | 与基线公平对比 |
| 第 8 周 | 最终集成 | 报告、幻灯片、演示视频、可复现材料 |

## 6. 评价建议

建议评价维度：

- 25% 可复现性与工程质量；
- 25% 基线与改进方法对比质量；
- 20% 失败或约束违约分析深度；
- 30% 最终报告、演示和技术解释。

## 7. 最终提交

最终提交包应包含：

- 源代码或可运行脚本；
- 算例文件或生成脚本；
- 求解器/模型配置与随机种子；
- 目标值、可行性、运行时间和违约表；
- 路线图或文本路线输出；
- 失败案例分析；
- 最终报告与 5-8 分钟演示视频。

## 8. 推荐第一步

先阅读 [SOTA and demo references](sota_demo_references.md)，选择 OR-Tools 或 PyVRP 作为第一个基线，然后完成 [Week 1 Lab](../labs/week01_setup.md)。一个可行路线和清楚日志，就是第一阶段里程碑。
